🔥 Firebase Integration Guide

Step-by-step guide to connect CropSense AI to Firebase — Authentication, Firestore Database & Storage

1 Create a Firebase Project

  • Go to console.firebase.google.com
  • Click "Add project"
  • Name it crop-disease-detection
  • Disable Google Analytics (optional) → Click "Create project"
  • Wait for it to load → Click "Continue"

2 Register Your Web App

  • In Firebase Console → Click the Web icon </>
  • App nickname: CropSense Web
  • Check "Also set up Firebase Hosting" (optional)
  • Click "Register app"
  • You'll see a config object — copy it
⚠️ Keep your config safe. Never share your API key publicly on GitHub without environment variables.

3 Paste Config into firebase-config.js

Open js/firebase-config.js and replace the placeholder values:

const firebaseConfig = {
  apiKey:            "AIzaSy...",           // ← paste yours
  authDomain:        "yourproject.firebaseapp.com",
  projectId:         "yourproject",
  storageBucket:     "yourproject.appspot.com",
  messagingSenderId: "123456789",
  appId:             "1:123456:web:abc123"
};

4 Enable Firebase Authentication

  • Firebase Console → Left menu → Authentication
  • Click "Get started"
  • Under Sign-in method tab → Click Email/Password
  • Toggle Enable → Click Save
✅ This handles user registration and login securely.

5 Create Firestore Database

  • Firebase Console → Left menu → Firestore Database
  • Click "Create database"
  • Select "Start in test mode" (for development)
  • Choose your region (e.g., asia-south1 for India) → Click Done

│               └── createdAt:  Timestamp

Firestore Security Rules

Go to Firestore → Rules tab → paste these rules:

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Users can only read/write their own data
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;

      // Users can only access their own predictions
      match /predictions/{predId} {
        allow read, write: if request.auth != null && request.auth.uid == userId;
      }
    }
  }
}

Click Publish

6 Enable Firebase Storage

  • Firebase Console → Left menu → Storage
  • Click "Get started"
  • Start in test mode → Click Done

Storage Security Rules

Go to Storage → Rules tab → paste:

rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {

    // Profile pictures — only owner can upload
    match /avatars/{userId}.jpg {
      allow read: if true;
      allow write: if request.auth != null && request.auth.uid == userId;
    }

    // Prediction thumbnails — only owner can upload
    match /predictions/{userId}/{fileName} {
      allow read: if true;
      allow write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Click Publish

7 Update Your HTML Pages

Replace the old script tags in your HTML pages with the new Firebase versions:

In login.html — replace bottom scripts:

<!-- REMOVE this -->
<script src="../js/auth.js"></script>

<!-- ADD this instead -->
<script type="module" src="../js/auth-firebase.js"></script>

In signup.html — replace bottom scripts:

<!-- REMOVE this -->
<script src="../js/auth.js"></script>

<!-- ADD this instead -->
<script type="module" src="../js/auth-firebase.js"></script>

In dashboard.html — replace bottom scripts:

<!-- REMOVE these -->
<script src="../js/auth.js"></script>
<script src="../js/dashboard.js"></script>

<!-- ADD these instead -->
<script type="module" src="../js/dashboard-firebase.js"></script>

In predict.html — replace bottom scripts:

<!-- REMOVE these -->
<script src="../js/auth.js"></script>
<script src="../js/predict.js"></script>

<!-- ADD these (keep model.js and chatbot.js) -->
<script src="../js/model.js"></script>
<script type="module" src="../js/predict-firebase.js"></script>
<script src="../js/chatbot.js"></script>
⚠️ Important: Firebase JS files use type="module". This means they only work on a server (localhost or deployed). They will NOT work by opening HTML files directly (file://).

8 Test Locally with VS Code Live Server

  • Install VS Code if not installed
  • Install the "Live Server" extension in VS Code
  • Open your project folder in VS Code
  • Right-click pages/index.html → "Open with Live Server"
  • It opens at http://127.0.0.1:5500/pages/index.html
✅ Firebase modules work correctly on localhost!

9 Deploy to Firebase Hosting (Optional)

Run these commands in PowerShell from your project folder:

# Install Firebase CLI
npm install -g firebase-tools

# Login to Firebase
firebase login

# Initialize hosting in your project folder
firebase init hosting

# When asked:
# - Public directory: .  (dot = current folder)
# - Single-page app: No
# - Overwrite index.html: No

# Deploy
firebase deploy

Your site will be live at: https://yourproject.web.app
